Une Pull Request est une demande faite par un contributeur à un projet pour que le propriétaire de ce projet fusionne ses modifications ou ses ajouts au code existant. Cela permet au propriétaire de revoir les modifications avant de les intégrer au projet.
Un fork est une copie d'un dépôt existant sur GitHub qui vous permet de travailler sur le code sans affecter le dépôt original. Vous pouvez modifier le code dans votre fork, le pousser sur votre propre branche et créer une Pull Request pour proposer les modifications au dépôt original.
Les Issues sont des rapports de bogues, des demandes de fonctionnalités ou des tâches à accomplir associées à un dépôt GitHub. Ils permettent aux contributeurs de signaler les problèmes ou de demander des modifications et aux propriétaires du projet de les suivre.
